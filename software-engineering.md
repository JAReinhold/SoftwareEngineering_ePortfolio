# Software Engineering Enhancement

## Artifact

Contact Service

---

## Overview

Contact Service is a Java application that manages contact records using object-oriented design principles. The project emphasizes data validation, encapsulation, and reliability through comprehensive JUnit testing. For my capstone, I revisited the original implementation and enhanced its maintainability, organization, and validation while improving the overall quality of the codebase through refactoring and expanded automated testing.

---

## Enhancement Narrative

Artifact Description

The artifact I selected for the Software Design and Engineering category is my Contact Service project from CS-320. The project allows users to create, update, and delete contacts while enforcing validation rules on the stored data. I originally wrote it back in 2025 as part of my software testing and quality assurance course.

Justification for Inclusion

I chose this artifact because it demonstrates several core software engineering skills, including object-oriented programming, validation, and unit testing. While the original project met the assignment requirements, it also gave me opportunities to improve the design and efficiency of the code. I would even say I didn’t really know what I was doing when I initially wrote it, so revisiting it now at the end of the degree program with a greater understanding was an added bonus.

For this enhancement, I replaced the ArrayList with a HashMap to improve performance, reduced repeated lookup logic by creating a reusable helper method, strengthened phone number validation, and expanded the JUnit test suite with a few more tests. I remember edge cases being a part of my feedback in that course, so I wanted to touch on it a bit. These improvements made the code cleaner and more reliable. It also reduced the amount of code that was even necessary to achieve the same goal.

Course Outcome Coverage

This artifact helped me demonstrate my ability to design and evaluate computing solutions while considering performance and maintainability. It also supports the security outcome by improving validation and error handling to prevent invalid data from entering the system, where I had left some openings in my original submission.

Reflection

Working on this enhancement reminded me that there is often a difference between code that works and code that is well designed. The original project functioned correctly and I had gotten good marks on it, but revisiting it gave me the opportunity to improve its efficiency and structure. One challenge was making changes without breaking existing functionality, which reinforced the importance of testing. I also struggled with getting this to run again because I was importing it incorrectly in a sort of code review format in IntelliJ. I eventually realized I would have to create an entirely new project, and import all of the code into the IDE for the Junit tests to function correctly. I would say that this enhancement helped me better understand how small design decisions can have a meaningful impact on the quality of a software project. Going this far back in my degree program and working on something has also given me a bit of real world

experience in the aspect of working on someone else’s code and not knowing what is what, in this case just due to how long it’s been since I’ve worked on this, the concept still applies I believe.
